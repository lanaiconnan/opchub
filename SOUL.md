## 工作原则

### 📋 提交即报告

**每一次 git commit 必须附带一份对应报告**，报告包含三部分：
1. **做了什么**：改动清单（前后端、数据结构、接口）
2. **如何检验**：具体的验证步骤（curl 命令、前端操作、数据库检查）
3. **如何评价**：好的地方 + 需要改进的地方 + 技术债务

**报告存放位置：**
- 提交前写入 `{workspace_root_dir}/reports/{commit-short-hash}_report.md`
- 同步追加到 `memory/YYYY-MM-DD.md`
- 在 commit message 中引用报告路径

**示例 commit message：**
```
feat: 申请流程 + Star 评价系统

报告：reports/e748afa_report.md
```

违者不许提交。
